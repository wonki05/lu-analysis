## 6. 사용자가 직접 확인해야 하는 항목(근거 포함)

1. **2025 지문 참조어구 밑줄**(보고서도 '사용자 확인 필요'로 지정)
   - 대상: [01-03] ㉠피에 굶주린 형법전 / ㉡자연의 제일법칙 / ㉢교수대에 낭비된 감수성 / ㉣문학적 공범자 / ㉤봉쇄된 정의, [19-21] ㉠엄격한 기준, [25-27] ㉠트랜잭션의 동시성 제어.
   - 이유: 2025 PDF는 한글이 공백 없는 런으로 추출되어 밑줄 하위 범위 자동확정이 어려움. 임의 범위 지정 위험이 있어 미적용. 원문 대비 밑줄 범위를 확정해 반영 권장.

2. **2024 박스형 강조 반영 여부**
   - PDF 렌더 결과 Q5 `k-익명성`이 박스 처리인데 마크다운은 `<kbd>` 0개. 박스 강조를 전수 `<kbd>`로 반영할지 정책 결정 필요(보고서 미지적 항목).

3. **2016 정당론 밑줄 확장 후보**
   - `<u>㉠ 정당의…강화해야 한다</u>는 주장을`에서 '는 주장을' 포함 여부. 추출기는 확장 제시, 보고서 미기재 → 시각 재확인 후 결정 권장(미적용).

4. **2022·2023·2026 지문 참조어구 밑줄(부정어 stem 외)**
   - 이미지기반 PDF로 텍스트 추출 불가. 부정어 stem은 렌더 확인 후 복원 완료했으나, 지문 내 ㉠~ 참조어구 밑줄의 추가 복원은 마커별 개별 시각확인이 필요(보고서는 해당 연도에서 stem 부정어를 주 지적).

5. **2012 마커 표기 규약 혼재(원본)**
   - 원본부터 마커-안 14 / 마커-밖 5로 혼재. 원본 보존 원칙상 미변경. 통일 원하면 사용자 결정 필요.

6. **엄격 재현 기준의 조사·어미 경계(2012·2013·2017·2018·2019 일부)**
   - 자동 매칭에서 제외된 일부 마커(범위차/문학지문 등)는 후속 정밀 대조 권장.

## 8. 사용한 주요 도구·방법

- **PyMuPDF(fitz) 1.28.0**: 페이지 텍스트/단어 bbox 추출, 밑줄 벡터(수평 세그먼트) 검출, 페이지 렌더링(PNG).
- **밑줄 추출 파이프라인**(`ul.py`): 수평 벡터 세그먼트 ↔ 단어 bbox 매핑으로 밑줄 텍스트·범위 추출. 페이지 furniture(홀수형/성명/페이지번호)·표 그리드라인 필터. 2015 보고서 목록과 1:1 대조로 검증.
- **부정어 감사**(`audit_neg.py`): 전 연도 "부정어+것은?" 출현이 100% stem임을 확인(오탐 0). PDF 밑줄 수 == 미처리 stem 수 1:1 검증.
- **마커 병합 생성**(`gen_markers.py`) + **안전 적용기**(`apply_ul.py`): 공백무시 매칭 + 유일성 검사 + 태그 균형 확인. 다중라인 밑줄은 렌더 시각확인.
- **범위 확장기**(`extend_ul.py`): `</u>` 밖 조사를 밑줄 안으로 이동(2016).
- **poppler**(pdftotext/pdftoppm/pdfinfo) 보조.
- 전 과정 UTF-8 파일 입출력으로 Windows 콘솔 인코딩 문제 회피.
