## 6. 판단이 필요했던 부분

| 위치 | 선택한 처리 방식 | 그렇게 처리한 이유 | 가능한 대안 |
|---|---|---|---|
| 전체 문항 수 | `## 01`부터 `## 35`까지 35문항으로 유지 | 원본 PDF의 안내문과 본문이 35문항 구성임을 확인했으며, 2018학년도 원문 보존이 우선임 | 일반 지침의 30문항 기준에 맞추어 30번까지만 검수할 수도 있으나, 원본 누락이 발생하므로 부적절함 |
| [07-09] 공통 제시문, `<그림 1>`, `<그림 2>` | 캡션은 Markdown 텍스트로 전사하고 도식만 PNG로 분리 | 원문 캡션은 텍스트로 정확히 전사 가능하며, 이미지 파일은 문제 풀이에 필요한 시각 자료 자체만 포함해야 함 | 캡션까지 이미지에 포함할 수 있으나, 중복 전사 및 crop 불필요 확장이 발생함 |
| [16-18] 공통 제시문 | “해석”의 박스 표시를 `<kbd>해석</kbd>`로 보완 | 원본 페이지에서 “바로 해석의 몫이다”의 “해석”에 작은 박스 표시가 있음을 확인했으므로 Markdown 특수 표기로 반영함 | 박스 없이 일반 텍스트로 둘 수 있으나, 문항 16에서 참조되는 표지이므로 보존이 더 적절함 |
| 원문 내 작은 박스 표지 | `<kbd>...</kbd>` 사용 | GitHub Markdown에서 박스형 표지를 비교적 자연스럽게 표시할 수 있음 | HTML span 또는 일반 텍스트 사용 가능하나, 시각 표지 보존성이 떨어짐 |
| 원문 내 밑줄 표지 | `<u>...</u>` 사용 | 문항 참조 대상이므로 밑줄 정보를 유지해야 함 | Markdown 강조나 일반 텍스트 처리가 가능하나, 원문 시각 표지를 보존하지 못함 |
