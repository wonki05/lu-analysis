## 6. 판단이 필요했던 부분

| 위치 | 선택한 처리 방식 | 이유 | 가능한 대안 |
|---|---|---|---|
| 전체 문항 수 | 01번부터 35번까지 모두 유지 | 원본 PDF가 35문항으로 구성되어 있어 PDF 원문 기준으로 검수함. 사용자 지침의 일부 체크리스트에는 30문항 기준 표현이 있으나, 해당 연도 원문은 35문항이다. | 30번까지만 유지하는 방식은 원문 문제지를 누락하게 되므로 채택하지 않음. |
| 33～35번 제시문 및 35번 <보기>의 영문 제목 꺾쇠괄호 | `\<The Price Is Right\>`, `\<Open Your Heart\>`로 Markdown escape 적용 | GitHub 렌더링에서 영문 꺾쇠괄호 표현이 HTML 태그처럼 해석되어 사라지는 것을 방지하기 위함. 렌더링 결과는 원문처럼 `<The Price Is Right>`, `<Open Your Heart>`로 보인다. | 원문 그대로 `<The Price Is Right>`와 `<Open Your Heart>`를 입력할 수 있으나 GitHub 렌더링 안정성이 떨어짐. |
| 12번 <그림> | 표제와 그림 제목은 Markdown 텍스트로 두고 도식만 PNG 처리 | 표제·제목은 텍스트로 정확히 전사 가능하고, 이미지에는 흐름도 자체만 포함하는 것이 crop 지침에 부합함. | 표제와 제목까지 이미지에 포함할 수 있으나 불필요한 텍스트 이미지화가 됨. |
