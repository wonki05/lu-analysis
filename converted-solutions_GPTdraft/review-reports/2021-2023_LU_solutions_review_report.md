# LEET 언어이해 공식 해설 Markdown 검수 보고서 (2021-2023)

## 0. 검수 개요

- 원문 비교대상: `법전협_2021 LEET 해설_언어이해.pdf`, `법전협_2022 LEET 해설_언어이해.pdf`, `법전협_2023 LEET 해설_언어이해.pdf`
- 검수대상: `2021_LU_official+solutions.md`, `2022_LU_official+solutions.md`, `2023_LU_official+solutions.md`
- PDF 페이지 수: 2021년 66쪽, 2022년 74쪽, 2023년 80쪽
- Markdown 문항 수: 세 파일 모두 01-30번 30개 문항 존재
- 검수 방법: PDF를 페이지 이미지로 렌더링한 뒤, 문항별 해설 표기ㆍ출제방향ㆍ정답 표기ㆍ그래프/표 참조 부분을 원문 이미지 및 OCR 텍스트와 대조하였다. PDF 자체가 스캔 이미지 기반이라 OCR 오차가 있으므로, 주요 오류는 이미지 렌더링을 다시 확인하여 확정하였다.

## 1. 2021학년도 LU

### 1.1 검토요약

- 문항 번호는 01-30까지 모두 존재한다.
- 문제 원문/제시문/선택지는 Markdown에 포함되어 있지 않아, “문제 원문은 포함하지 않는다”는 지시에는 부합한다.
- 해설 본문은 대체로 자연스럽게 정리되어 있으며, 2023년 파일에서 보이는 대규모 OCR 잔재는 확인되지 않았다.
- 다만 원문 PDF의 문항별 해설 표에 존재하는 `정답 : ①` 형식의 정답 행이 Markdown에는 전부 빠져 있다. 이는 모든 문항의 해설 구성상 중요한 누락이다.

### 1.2 오탈자 검수결과

- 본문 해설의 큰 의미를 바꾸는 명백한 오탈자는 확인하지 못했다.
- 그러나 공식 해설의 메타 정보가 누락되거나 변형된 부분이 있다.
  - **전 문항 공통:** PDF의 `문제 풀이  정답 : ...` 행이 Markdown에 없음.
  - **전 문항 공통:** PDF의 `정답 해설`, `오답 해설` 라벨이 Markdown에서는 `### 선택지별 해설` 아래의 선택지별 설명으로 통합되어, 원문에 있던 라벨 글자 자체는 보존되지 않았다.
  - **출제방향:** 원문은 `4. 난이도 및 출제 시 유의점`으로 되어 있으나, Markdown은 `## 4. 난이도`와 `## 5. 문항 출제시 유의점 및 강조점`으로 분리되어 있다. 내용 자체는 대체로 들어 있으나 제목/번호 구조가 원문과 다르다.

### 1.3 이미지(그래프 등) 검수결과

- Markdown 파일 내 이미지 링크는 없다.
- 원문 해설의 본문 부분에는 별도 삽입 이미지나 그래프가 독립적으로 들어간 것으로 보이지 않는다.
- 다만 27번 해설은 `<보기>`의 S자 곡선 그래프를 전제로 설명한다. 문제 원문 제외 지시에 따르면 그래프 미첨부는 허용 가능한 처리이나, AI 분석 목적상 그래프 원문 없이 해설만으로는 맥락이 부족할 수 있다.

### 1.4 표 검수결과

- 원문 PDF의 문항별 정보는 파란색/회색 라벨 표 형태로 배치되어 있다. Markdown은 이를 `문항구분`, `평가 목표`, `문항 해설`, `선택지별 해설` 구조로 재구성하였다.
- 의미 있는 데이터 표나 계산표가 누락된 정황은 확인되지 않았다.
- 단, 위 표 구조에 포함되어 있던 `정답 : ...`, `정답 해설`, `오답 해설`은 텍스트 정보이므로 단순 레이아웃이 아니라 내용 누락으로 보아야 한다.

### 1.5 잘못되었다고 보이는 부분 개선안

- **OCR/시각 대조 기반 개선안**
  - 각 문항의 `### 문항 해설` 직후 또는 `### 선택지별 해설` 직전에 원문 PDF의 정답 행을 추가한다.
  - 권장 형식: `정답 : (1)`처럼 현재 2022년 파일이 사용하는 형식으로 통일한다.
  - 원문 라벨을 보존하려면 각 문항에서 정답 선택지 설명 앞에 `#### 정답 해설`, 나머지 선택지 설명 앞에 `#### 오답 해설`을 추가한다.
- **원문 제목 대조 기반 개선안**
  - 출제방향의 4번 제목을 원문에 맞추려면 `## 4. 난이도 및 출제 시 유의점`으로 병합한다.
  - 만약 가공 규칙상 4번/5번을 분리해야 한다면, 보고서 또는 README에 “구조화 편의를 위해 원문 제목을 분리함”이라고 명시하는 편이 안전하다.

### 1.6 사용자가 직접 확인했으면 하는 부분

- 2021년 파일은 정답 번호가 전 문항 누락되어 있으므로, 업로드 전 PDF의 `정답 : ...` 행을 다시 대조해 일괄 삽입하는 것을 권장한다.
- 27번처럼 그래프를 참조하는 해설은 문제 원문 제외 정책과 AI 분석 목적 사이의 균형을 다시 결정할 필요가 있다.

## 2. 2022학년도 LU

### 2.1 검토요약

- 문항 번호는 01-30까지 모두 존재한다.
- `정답 : (n)` 행은 30개 문항 모두에 존재한다.
- 해설 본문은 전반적으로 가장 안정적이다.
- 문제 원문은 포함되어 있지 않아 지시사항에 부합한다.

### 2.2 오탈자 검수결과

- 의미를 바꾸는 명백한 오탈자는 발견하지 못했다.
- 다만 원문 형식 대비 다음 구조 변형이 있다.
  - **전 문항 공통:** 원문 PDF의 `정답 해설`, `오답 해설` 라벨이 Markdown에서는 `### 선택지별 해설`로 통합되어 있다. 정답 번호는 보존되어 있으므로 2021ㆍ2023년에 비해 정보 손실은 작지만, 라벨 글자 자체는 누락되어 있다.
  - **출제방향:** 원문 PDF는 `4. 난이도 및 출제 시 유의점`이라는 단일 항목인데, Markdown은 `## 4. 난이도`와 `## 5. 문항 출제시 유의점 및 강조점`으로 분리되어 있다.

### 2.3 이미지(그래프 등) 검수결과

- Markdown 파일 내 이미지 링크는 없다.
- 해설 본문 자체에 독립 그래프/그림이 누락된 정황은 확인되지 않았다.
- 11번 해설은 `<보기>`의 그래프/자료를 전제로 설명한다. 문제 원문 제외 지시상 그래프가 빠진 것은 허용 가능한 처리이나, 독립 분석용 데이터로 사용할 때는 원문 그래프를 별도로 확인해야 한다.

### 2.4 표 검수결과

- 의미 있는 표 누락은 확인되지 않았다.
- 공식 해설의 메타 정보 표는 Markdown heading/list 구조로 변환되어 있다.
- `정답 : (n)` 행은 보존되어 있어 2021ㆍ2023년보다 상태가 좋다. 다만 `정답 해설`/`오답 해설` 라벨은 보존되지 않았다.

### 2.5 잘못되었다고 보이는 부분 개선안

- **원문 형식 대조 기반 개선안**
  - 각 문항의 정답 선택지 설명 앞에 `#### 정답 해설`, 오답 선택지 설명 앞에 `#### 오답 해설`을 추가하면 원문 정보 보존성이 올라간다.
  - 출제방향 제목을 원문과 맞추려면 `## 4. 난이도 및 출제 시 유의점`으로 병합한다.
- **추론상 개선안**
  - 현재 2022년 파일의 `정답 : (n)` 표기는 2021ㆍ2023년 파일의 보정 기준 형식으로 활용하기 적합하다.

### 2.6 사용자가 직접 확인했으면 하는 부분

- 원문 제목 구조까지 정확히 보존할 것인지, 아니면 저장소 내 통일성을 위해 4번/5번 제목을 분리할 것인지 결정해야 한다.
- 11번의 그래프 참조가 AI 분석에 필요한 경우, 문제 원문 제외 원칙을 유지하되 그래프만 별도 asset으로 둘지 검토할 수 있다.

## 3. 2023학년도 LU

### 3.1 검토요약

- 문항 번호는 01-30까지 모두 존재한다.
- 문제 원문은 포함되어 있지 않아 지시사항에 부합한다.
- 그러나 2023년 Markdown은 세 파일 중 오류와 OCR 잔재가 가장 많다.
- 특히 `정답 : ...` 행이 전 문항에서 누락되어 있고, 출제방향에는 원문과 다른 단어가 들어간 곳이 확인된다.
- 단어 내부 공백, 문장 중간의 불필요한 줄바꿈, 곧장 붙어야 할 어절 분리, 인용부호 혼용이 다수 존재한다.

### 3.2 오탈자 검수결과

#### 확정 오류

1. **출제방향 3번, 인문 분야 설명**
   - 현재 Markdown: `‘낭만주의, ‘낭만적인 것’, ‘예술’, ‘종교’ 등의 개념`
   - 원문 PDF 확인: `‘낭만주의’, ‘낭만적인 것’, ‘예술’, ‘철학’ 등의 개념`
   - 개선안: `‘낭만주의’, ‘낭만적인 것’, ‘예술’, ‘철학’ 등의 개념`
   - 판정 근거: 원문 렌더링 이미지와 OCR을 다시 대조한 결과이므로 **OCR/시각 대조 기반 수정**이다.

2. **출제방향 3번, 규범 분야 설명**
   - 현재 Markdown: `판사의 판결에 진솔함이 요구되는 이유와 관련된 의문을 다루는 제시문`
   - 원문 PDF 확인: `판사의 판결에 진솔함이 요구되는 이유와 관련된 논의를 다루는 제시문`
   - 개선안: `판사의 판결에 진솔함이 요구되는 이유와 관련된 논의를 다루는 제시문`
   - 판정 근거: 원문 렌더링 이미지와 OCR을 다시 대조한 결과이므로 **OCR/시각 대조 기반 수정**이다.

3. **출제방향 5번 제목**
   - 현재 Markdown: `## 5. 문항 출제시 유의점 및 강조점`
   - 원문 PDF 확인: `5. 출제 시 유의점`
   - 개선안: `## 5. 출제 시 유의점`
   - 판정 근거: 원문 렌더링 이미지 대조에 따른 **원문 대조 기반 수정**이다.

4. **전 문항 공통 정답 행 누락**
   - 현재 Markdown에는 01-30번 모두 `정답 : (n)` 행이 없다.
   - 원문 PDF의 문항별 풀이 표에는 `문제 풀이  정답 : ...` 행이 존재한다.
   - 개선안: 2022년 파일과 동일한 형식으로 각 문항에 `정답 : (n)` 행을 삽입한다.
   - 판정 근거: 원문 PDF의 문항별 해설 표를 시각 대조한 결과이므로 **원문 대조 기반 수정**이다.

#### OCR 잔재 및 한글 어법상 부자연스러운 부분

아래는 대표 예시이며, 2023년 파일 전체에 유사 패턴이 반복된다.

| 위치 | 현재 Markdown | 개선안 | 구분 |
|---|---|---|---|
| 01번 평가 목표 | `거짓말하` / 빈 줄 / `기를 선택하는... 정확` / 빈 줄 / `하게 파악하고...` | `거짓말하기를 선택하는 ... 정확하게 파악하고...` | OCR 줄바꿈 정리 |
| 01번 문항 해설 | `판 사에게`, `반 대론`, `제기했 다`, `취지` / 빈 줄 / `로 끝맺고` | `판사에게`, `반대론`, `제기했다`, `취지로 끝맺고` | OCR 공백 정리 |
| 03번 평가 목표 | `통제가 미쳐야 한다고 는 주장`, `바탕 으로`, `위해설계된` | `통제가 미쳐야 한다는 주장`, `바탕으로`, `위해 설계된` | OCR 공백 및 띄어쓰기 정리 |
| 05번 문항 해설 | `현상적 의식은` / 빈 줄 / `감응력이 있는...` | 한 문단으로 결합 | OCR 줄바꿈 정리 |
| 08번 문항 해설 | `리보` / 빈 줄 / `솜에서 합성되는...` | `리보솜에서 합성되는...` | OCR 줄바꿈 정리 |
| 30번 문항 해설 | `데리다의 견` / 빈 줄 / `해 사이의...` | `데리다의 견해 사이의...` | OCR 줄바꿈 정리 |

추가로 2023년 파일에는 직선 인용부호 `'`, `"`가 다량 섞여 있다. 2021ㆍ2022년 파일은 주로 `‘ ’`, `“ ”`를 쓰고 있어, 2023년의 직선 인용부호는 OCR 잔재일 가능성이 높다. 원문 해설의 인용부호 체계를 기준으로 통일하는 것이 좋다.

### 3.3 이미지(그래프 등) 검수결과

- Markdown 파일 내 이미지 링크는 없다.
- 해설 본문 자체에 독립 삽입 이미지가 누락된 정황은 확인하지 못했다.
- 27번 해설은 `<보기>`의 그래프를 직접 참조한다. 문제 원문 제외 지시상 그래프 미첨부는 허용 가능한 처리이나, 그래프가 없으면 해설의 일부 추론 맥락은 독립적으로 재현하기 어렵다.

### 3.4 표 검수결과

- 의미 있는 데이터 표 누락은 확인되지 않았다.
- 공식 해설의 메타 정보 표는 Markdown 구조로 변환되어 있다.
- 다만 이 표에 들어 있던 `정답 : ...` 행이 전 문항에서 누락되어 있으므로, 표 검수 관점에서도 중요한 누락으로 보아야 한다.
- `정답 해설`/`오답 해설` 라벨 역시 별도로 보존되지 않았다.

### 3.5 잘못되었다고 보이는 부분 개선안

- **OCR/시각 대조 기반 개선안**
  1. 출제방향의 `종교`를 `철학`으로 수정한다.
  2. 출제방향의 `의문`을 `논의`로 수정한다.
  3. `## 5. 문항 출제시 유의점 및 강조점`을 `## 5. 출제 시 유의점`으로 수정한다.
  4. 각 문항에 PDF의 정답 행을 추가한다.
  5. 단어 내부 공백과 문장 중간 줄바꿈을 전수 정리한다. 특히 `판 사`, `반 대론`, `사 법`, `정 당성`, `도 덕`, `법 적`, `리보\n솜`, `견\n해` 등은 한글 어법상 명백히 이어져야 한다.
- **추론상 개선안**
  - 2023년 파일은 부분 수정만으로도 사용할 수는 있으나, 오류 패턴이 광범위하므로 원문 PDF에서 다시 OCR한 뒤 2022년 파일의 포맷 규칙에 맞춰 재생성하는 편이 안전하다.

### 3.6 사용자가 직접 확인했으면 하는 부분

- 2023년 파일은 정답 행과 OCR 잔재가 동시에 문제되므로, GitHub 업로드 전 전 문항을 한 번 더 재대조하는 것을 권장한다.
- `정답 해설`/`오답 해설` 라벨을 보존할지, 아니면 현재처럼 선택지별 설명으로 통합할지 저장소 표준을 정해야 한다.
- 27번 그래프 참조는 문제 원문 제외 정책과 별도로, AI가 해설을 독립적으로 이해해야 하는지 여부에 따라 asset 처리 여부를 결정해야 한다.

## 4. 공통 개선 권고

1. **정답 행 표준화**
   - 2022년 파일 형식인 `정답 : (n)`을 2021년ㆍ2023년 파일에도 동일하게 적용한다.

2. **원문 라벨 보존**
   - 공식 해설의 `문제 풀이`, `정답 해설`, `오답 해설` 라벨은 단순 장식이 아니라 해설 구조를 나타내는 텍스트이다.
   - 최소한 `정답 해설`/`오답 해설`은 Markdown heading으로 보존하는 것이 바람직하다.

3. **출제방향 제목 원문성 유지**
   - 2021ㆍ2022년: `4. 난이도 및 출제 시 유의점`
   - 2023년: `4. 난이도`, `5. 출제 시 유의점`
   - 저장소 통일을 위해 제목을 재구성할 수는 있으나, 그 경우 원문과 달라졌음을 명시해야 한다.

4. **그래프 참조 문항 처리 원칙 명시**
   - 문제 원문 제외 원칙 때문에 그래프를 넣지 않는 것은 이해되지만, 2021년 27번, 2022년 11번, 2023년 27번처럼 해설이 그래프를 전제로 하는 문항은 README나 메타데이터에 “문제 원문 그래프는 제외됨”이라고 명시하는 것이 좋다.

5. **2023년 재정제 우선순위**
   - 1순위: `정답 : ...` 행 삽입
   - 2순위: 출제방향 확정 오류 수정
   - 3순위: 단어 내부 공백/줄바꿈 전수 정리
   - 4순위: 인용부호 체계 통일
