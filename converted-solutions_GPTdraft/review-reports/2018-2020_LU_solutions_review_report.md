# LEET 언어이해 공식 해설 Markdown 변환 검수 보고서 (2018-2020)

## 0. 검수 범위 및 방법

### 검수 대상

| 연도 | 원문 PDF | 검수 Markdown | PDF 쪽수 | Markdown 문항 수 |
|---:|---|---|---:|---:|
| 2018 | `법전협_2018 LEET 해설_언어이해.pdf` | `2018_LU_official+solutions.md` | 74쪽 | 35문항 |
| 2019 | `법전협_2019 LEET 해설_언어이해.pdf` | `2019_LU_official+solutions.md` | 64쪽 | 30문항 |
| 2020 | `법전협_2020 LEET 해설_언어이해.pdf` | `2020_LU_official+solutions.md` | 72쪽 | 30문항 |

- 첨부된 검수 대상은 세 연도 모두 `.md` 파일이었다. 이미지 포함 패키지 형태의 `.zip` 파일은 첨부되어 있지 않았다.
- 문제 원문은 Markdown 변환 대상에서 제외하라는 전제에 맞추어, PDF의 문제 본문·선택지·제시문 영역은 내용 누락 판정에서 제외하였다.
- 다만 해설 본문이 문제 원문 속의 `<그림>`, `<표>`, `〈그림〉`을 참조하는 경우에는 “직접 확인 권고” 항목에 별도로 표시하였다.

### 검수 방법

- PDF 전 페이지를 이미지로 렌더링한 뒤 원문 레이아웃, 해설 박스, 그림·표 존재 여부를 확인하였다.
- 2018·2019 PDF는 이미지 기반 스캔이어서 텍스트 추출이 되지 않았고, 2020 PDF의 내장 OCR도 오인식이 많았다. 따라서 OCR은 보조적으로만 사용하고, 문제가 의심되는 부분은 렌더링 이미지로 대조하였다.
- Markdown의 문항 번호, 정답 행, 이미지 참조, 표 문법, 강조 표기, `문항유형`·`내용영역` 포함 여부를 기계적으로 점검하였다.

### 전체 구조 점검 결과

| 연도 | 문항 번호 연속성 | 정답 행 수 | 이미지 참조 | Markdown 표 문법 | 볼드/밑줄 표기 | 구조상 주요 판정 |
|---:|---|---:|---:|---:|---:|---|
| 2018 | `01`-`35` 정상 | 35 | 0 | 0 | 0 | 전반 양호. 제목 1곳 수정 권고. |
| 2019 | `01`-`30` 정상 | 30 | 0 | 0 | 0 | 전반 양호. `문항유형 :` 라벨의 체계적 누락. |
| 2020 | `01`-`30` 정상 | 30 | 0 | 0 | 0 | 본문은 대체로 양호하나, 모든 문항의 `내용영역` 및 `문항유형 :` 라벨 누락. |

## 1. 2018학년도 언어이해

### 1.1 검토요약

- Markdown에는 `01`부터 `35`까지 모든 문항 해설이 존재하고, 각 문항의 정답 행도 35개가 모두 존재한다.
- 해설 본문에서 문장 단위의 대규모 누락, 선택지별 해설 누락, 수식·기호의 명백한 누락은 발견되지 않았다.
- 원문 PDF의 해설 박스 안에는 독립적인 그림·그래프·표 이미지가 확인되지 않았다. Markdown에 이미지 참조가 없는 것은 해설 범위 기준으로는 문제되지 않는다.
- 다만 출제방향 5번 제목에서 원문과 다른 표기가 발견되었다.
- `DNA를 활용한 사진법 계산`은 표현상 어색하지만, PDF 원문 이미지에서 동일하게 확인되므로 수정 대상이 아니다.

### 1.2 오탈자 검수결과

| 위치 | Markdown 현재 표기 | 원문 기준 개선안 | 판단 근거 | 구분 |
|---|---|---|---|---|
| 출제방향 5번 제목 | `## 5. 문항 출제시 유의점 및 강조점` | `## 5. 문항 출제 시의 유의점 및 강조점` | 원문 PDF 렌더링 이미지에서 `문항 출제 시의 유의점 및 강조점`으로 확인 | OCR 보조 + 시각 확인 |

추가로, 대제목 `# 출제방향` 및 `## 1. 출제의 기본방향`은 원문 제목인 `2018학년도 언어이해 영역 출제 방향`, `출제의 기본 방향`을 저장소용으로 간소화한 형태로 보인다. 엄격한 원문 재현을 요구한다면 제목도 원문형으로 복원하는 것이 좋다.

### 1.3 이미지(그래프 등) 검수결과

- Markdown 내부 이미지 참조: 없음.
- 해설 박스 내부의 독립 이미지: 발견되지 않음.
- 문제 원문 영역에는 DNA 컴퓨팅 관련 그래프가 존재하고, 해설 본문은 `<그림 2>`를 참조한다. 그러나 해당 그래프는 문제 원문 영역의 자료이므로, “문제 원문 제외” 전제에서는 누락으로 보지 않았다.

### 1.4 표 검수결과

- Markdown 표 문법으로 변환된 표: 없음.
- 해설 박스 내부의 실질적 데이터 표: 발견되지 않음.
- 해설 본문 중 베나타 문항에서 `<표>`를 참조하는 부분이 있다. 해당 표는 문제 원문 자료로 판단되며, 해설 박스 내 독립 표 누락은 발견되지 않았다.

### 1.5 잘못되었다고 보여지는 부분 개선안

#### OCR 보조 + 원문 이미지 확인으로 얻은 개선안

```md
## 5. 문항 출제 시의 유의점 및 강조점
```

#### 추론상 개선안

- 없음. 해설의 설명 자체가 부적합하거나 한글 어법상 수정이 필요하다고 판단되는 부분은 발견되지 않았다.

### 1.6 사용자가 직접 확인하면 좋은 부분

- 해설만 단독으로 AI 분석에 사용할 경우, `<그림 2>` 및 `<표>` 참조가 문제 원문 자료를 가리키므로 문맥 정보가 부족할 수 있다. 현재 변환 정책상 문제 원문 제외가 맞지만, 향후 “해설 단독 완결성”을 높일 목적이라면 참조 그림·표만 별도 에셋으로 포함할지 정책 결정이 필요하다.

## 2. 2019학년도 언어이해

### 2.1 검토요약

- Markdown에는 `01`부터 `30`까지 모든 문항 해설이 존재하고, 각 문항의 정답 행도 30개가 모두 존재한다.
- 해설 본문에서 문장 단위의 대규모 누락, 선택지별 해설 누락, 수식·기호의 명백한 누락은 발견되지 않았다.
- 해설 박스 내부의 독립 이미지·그래프·표는 발견되지 않았다.
- 원문 해설 박스의 `문항 성격` 행은 `문항유형 : ...`과 `내용영역 : ...`으로 구성되어 있으나, Markdown에서는 `문항유형 :`이라는 글자가 모든 문항에서 생략되어 있다.
- 출제방향 5번 제목에서도 2018과 동일한 제목 표기 문제가 확인되었다.

### 2.2 오탈자 검수결과

| 위치 | Markdown 현재 표기 | 원문 기준 개선안 | 판단 근거 | 구분 |
|---|---|---|---|---|
| 출제방향 5번 제목 | `## 5. 문항 출제시 유의점 및 강조점` | `## 5. 문항 출제 시의 유의점 및 강조점` | 원문 PDF 렌더링 이미지에서 `문항 출제 시의 유의점 및 강조점`으로 확인 | OCR 보조 + 시각 확인 |
| 01-30 전 문항의 문항구분 | `* 문항 성격 : 주제, 구조, 관점 파악` 등 | `* 문항 성격 : 문항유형 : 주제, 구조, 관점 파악` 등 | 원문 해설 박스는 `문항 성격` 행 안에 `문항유형 : ...`을 표시함. Markdown에는 `문항유형` 문자열이 0회임 | OCR 보조 + 시각 확인 |

예시 수정안은 다음과 같다.

```md
### 문항구분

* 문항 성격 : 문항유형 : 주제, 구조, 관점 파악
* 내용영역 : 규범
* 평가 목표 : ...
* 정답 : (1)
```

현재 Markdown도 문항유형의 값 자체는 보존하고 있으나, 원문에 있는 `문항유형 :` 라벨이 빠져 있으므로 “모든 글자와 단어” 기준에서는 수정이 필요하다.

### 2.3 이미지(그래프 등) 검수결과

- Markdown 내부 이미지 참조: 없음.
- 해설 박스 내부의 독립 이미지: 발견되지 않음.
- 문제 원문 영역에는 시각 자료가 포함된 문항이 있으나, 문제 원문 제외 전제에서는 Markdown 누락으로 보지 않았다.

### 2.4 표 검수결과

- Markdown 표 문법으로 변환된 표: 없음.
- 해설 박스 내부의 실질적 데이터 표: 발견되지 않음.
- 원문 해설 박스의 좌측 라벨 영역은 표 형태 레이아웃이지만, Markdown에서는 제목·목록 구조로 변환되어 있다. 의미 정보는 대체로 보존되나, 원문 레이아웃을 엄격히 보존해야 한다면 `문항 성격`, `평가 목표`, `문제 풀이`, `정답 해설`, `오답 해설` 라벨의 보존 방식도 별도 기준화가 필요하다.

### 2.5 잘못되었다고 보여지는 부분 개선안

#### OCR 보조 + 원문 이미지 확인으로 얻은 개선안

1. 제목 수정

```md
## 5. 문항 출제 시의 유의점 및 강조점
```

2. 전 문항 `문항 성격` 행 수정

```md
* 문항 성격 : 문항유형 : [원래 Markdown에 적힌 문항유형 값]
* 내용영역 : [현재 Markdown의 내용영역 값]
```

#### 추론상 개선안

- 없음. 해설의 설명 자체가 부적합하거나 한글 어법상 수정이 필요하다고 판단되는 부분은 발견되지 않았다.

### 2.6 사용자가 직접 확인하면 좋은 부분

- 현재 Markdown은 원문 해설 박스의 `문제 풀이`, `정답 해설`, `오답 해설` 같은 박스 라벨을 `문항 해설`, `선택지별 해설` 등의 Markdown heading으로 재구성하였다. 정보 접근성 측면에서는 적절하지만, 원문 라벨의 완전 보존을 목표로 한다면 이 구조 변환을 허용할지 확인이 필요하다.

## 3. 2020학년도 언어이해

### 3.1 검토요약

- Markdown에는 `01`부터 `30`까지 모든 문항 해설이 존재하고, 각 문항의 정답 행도 30개가 모두 존재한다.
- 해설 본문에서 문장 단위의 대규모 누락, 선택지별 해설 누락, 수식·기호의 명백한 누락은 발견되지 않았다.
- 그러나 2020 Markdown은 모든 문항의 `문항구분`에서 원문 해설 박스의 `내용영역 : ...` 행을 누락하고 있다.
- 또한 2019와 마찬가지로 `문항유형 :` 라벨이 모든 문항에서 생략되어 있다.
- 출제방향 5번 제목에서도 원문과 다른 표기가 발견되었다.

| 출제방향 5번 제목 | `## 5. 문항 출제시 유의점 및 강조점` | `## 5. 문항 출제 시의 유의점 및 강조점` | 원문 PDF 렌더링 이미지에서 `문항 출제 시의 유의점 및 강조점`으로 확인 | OCR 보조 + 시각 확인 |

- 원문 해설 박스 자체의 좌측 라벨 영역은 표 형태 레이아웃이지만, Markdown에서는 제목·목록 구조로 변환되어 있다. 이 구조 변환은 정보 접근성 측면에서는 허용 가능하나, 원문 레이아웃 재현을 목표로 한다면 별도 기준이 필요하다.

### 3.5 잘못되었다고 보여지는 부분 개선안

#### OCR 보조 + 원문 이미지 확인으로 얻은 개선안

1. 제목 수정

```md
## 5. 문항 출제 시의 유의점 및 강조점
```

2. 전 문항에 `문항유형 :` 라벨 복원

```md
* 문항 성격 : 문항유형 : [현재 Markdown의 문항 성격 값]
```

3. 전 문항에 `내용영역 : ...` 행 추가

```md
* 내용영역 : [위 표의 문항 범위별 내용영역]
```

#### 추론상 개선안

- 2020의 `내용영역` 범위 표는 원문 대표 페이지 확인과 문항 세트 구성 대조를 함께 사용하여 작성하였다. 일괄 수정 전에는 원문 PDF의 각 세트 첫 문항 해설 박스에서 한 번 더 육안 확인하는 것을 권장한다.
- 해설의 설명 자체가 부적합하거나 한글 어법상 수정이 필요하다고 판단되는 부분은 별도로 발견되지 않았다.

### 3.6 사용자가 직접 확인하면 좋은 부분

- 2020 파일은 `내용영역`이 전부 빠져 있으므로, GitHub 업로드 전 일괄 보정이 필요하다.
- 2020 PDF의 내장 OCR은 오인식이 많으므로, 자동 OCR 결과만으로 수정하지 말고 렌더링 이미지 기준으로 `내용영역`을 최종 확인하는 것이 안전하다.

### 우선 수정 1순위

1. `2020_LU_official+solutions.md` 전 문항에 `내용영역 : ...` 추가.
2. `2020_LU_official+solutions.md` 전 문항의 `문항 성격` 행에 `문항유형 :` 라벨 복원.
3. `2019_LU_official+solutions.md` 전 문항의 `문항 성격` 행에 `문항유형 :` 라벨 복원.

### 우선 수정 2순위

1. 세 연도 모두 출제방향 5번 제목을 `문항 출제 시의 유의점 및 강조점`으로 수정.
2. 원문 제목까지 엄격히 보존할 경우, 각 파일의 대제목을 `[연도]학년도 언어이해 영역 출제 방향`으로 보강하고, `출제의 기본방향`을 `출제의 기본 방향`으로 띄어쓰기 수정.

### 이미지·표 관련 최종 판단

- 해설 박스 내부의 독립 이미지·그래프·실질 표 누락은 발견되지 않았다.
- 문제 원문 영역의 그림·표는 현재 변환 정책상 제외된 것으로 판단된다.
- 다만 해설이 문제 원문 속 그림·표를 참조하는 문항은 AI 분석 시 문맥 부족이 생길 수 있으므로, 향후 저장소 정책에 따라 “해설 참조용 이미지/표만 별도 에셋으로 포함”하는 방안을 검토할 수 있다.
